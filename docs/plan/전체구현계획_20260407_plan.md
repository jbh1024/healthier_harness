### 7-4. 게시판 첨부파일 MIME 검증 + 업로드/다운로드 엔드포인트 (SEC-06, BRD-03/11)
- **인프라**: `AttachmentValidator` — MIME(이미지 3종 + 문서 6종) + 확장자 + 크기(10MB) 검증

  - `BoardService.getAttachment`, `deleteAttachment`

    - `GET /board/{postId}/attachments/{id}` (FileSystemResource 다운로드, UTF-8 filename* 헤더)
    - `DELETE /board/{postId}/attachments/{id}`
  - `ErrorCode`: INVALID_ATTACHMENT, ATTACHMENT_TOO_LARGE, UNSUPPORTED_ATTACHMENT_TYPE, ATTACHMENT_LIMIT_EXCEEDED, ATTACHMENT_NOT_FOUND

### 7-5. 테스트 보강
- `AttachmentValidatorTest` (8 tests): 허용 이미지/문서, 빈 파일, 크기 초과, MIME/확장자 불일치, 경계값(정확히 10MB)
- `AuthServiceTest` 확장 (6 tests 추가): 재설정 요청/조용한 실패/정상 재설정/만료/사용 완료/잘못된 토큰
- 전체 테스트: **4 → 5 testsuite / 17 → 31 tests, 0 failed**

### 의사결정
- **이메일 서비스**: 외부 SDK 연동 없이 Console 로그 stub 제공. 운영 투입 시 `EmailService` 구현체만 교체 (예: Spring Mail, AWS SES)
- **비밀번호 재설정 토큰**: UUID 기반 32자, 30분 유효, 일회용. 기존 미사용 토큰은 새 요청 시 전부 삭제
- **재설정 후 기존 토큰 무효화**: 보안상 기존 로그인 세션은 모두 로그아웃 처리 (`refreshTokenRepository.deleteByUserId`)
- **존재하지 않는 이메일 재설정 요청**: 사용자 존재 여부 노출 방지를 위해 예외 없이 조용히 종료
- **게시판 첨부파일 API 분리**: 기존 `createPost`(JSON)과 첨부파일 업로드를 분리. 글 작성 → 첨부파일 업로드 2단계로 구성하여 단순성 확보
- **CORS 프로퍼티**: `allowed-origins`와 `allowed-origin-patterns` 둘 다 지원. prod에서 명시 안 하면 모든 origin 차단
