# 문서 작성

## 작성 원칙.

- 문서는 `docs/` 경로에 해당하는 문서에 내용을 작성한다. 
- 문서의 내용은 최대한 중복이 없이 간결하게 개조식으로 작성한다
- 구현 계획은 `docs/plan/요청내용_날짜_plan.md` 에 작성하고 매 구현 단계마다 문서를 업데이트 하도록 한다.
- `docs/PRD/PRD.md`에 있는 요구사항을 구현 할 때는 해당 `docs/PRD/요구사항ID.md` 을 만들고 작업 내용 및 의사결정, 작업 결과를 작성한다. 
- 그외에 지시한 작업은 지시한 내용부터 의사결정, 구현내용까지 정리해서 `docs/요청내용요약_날짜.md` 파일로 작성한다.
